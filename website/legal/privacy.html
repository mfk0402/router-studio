Privacy

Router Studio is a desktop application. This page summarizes how the product treats your data. For implementation details, see the Privacy section in the project README.

Local storage

Your OpenRouter API key and settings are stored on your computer. When the OS supports it, Electron’s safeStorage encrypts secrets with the platform keychain (macOS Keychain, Windows DPAPI, Linux libsecret where available).

What leaves your device

Code and context are sent to OpenRouter only when you explicitly use AI features (for example sending a chat message with project context). Routine file edits and local tooling do not upload your repository by default.

No telemetry

The app does not include analytics or a first-party cloud database. Third-party model providers (via OpenRouter) receive only the prompts and attachments you send as part of a request.

Optional account vault

If you use Router Studio’s optional local account feature, an encrypted vault may hold settings backups on disk. That vault is encrypted with your password and is not uploaded to Router Studio servers — there is no Router Studio-hosted account backend in the open-source distribution.

Some packaged builds enable email verification during registration: the app then contacts a verify URL you or your distributor configure (ROUTER_STUDIO_VERIFY_URL), which sends a one-time code. Only the email address is sent to that service for that purpose — not your OpenRouter key or vault contents. See the reference verify server.
